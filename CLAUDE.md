## Architecture

### Entry Flow
`index.html` → `src/main.tsx` → `src/App.tsx` (RouterProvider) → `src/routes.tsx` (23 routes)

### Routing
All routes are defined in `src/routes.tsx` using a flat structure under a single `Layout` wrapper. The `Layout` component (`src/components/Layout.tsx`) renders Header + Outlet + Footer with scroll-to-top on navigation.

Key routes: `/` (HomePageV3), `/solutions`, `/industries/*`, `/about`, `/process`, `/projects`, `/agents`, `/chatbots`, `/booking`, `/financial`. Multiple home page versions exist: V1 (`/home-v1`), V2 (`/home-v4`), V3 (`/` default).

### Component Organization
- `src/components/ui/` — shadcn/ui primitives (do not modify directly unless customizing the design system)
- `src/components/home/`, `homev2/` — Home page section variants
- `src/components/{feature}/` — Feature-specific sections (agents, chatbots, solutions, process, booking, ecommerce, fashion, travel, financial, projects, services, carousel)
- `src/components/shared/` — Reusable components (PageHeader, Layout)
- Page components live at `src/components/{PageName}.tsx` and compose feature section components

### Page Composition Pattern
Pages import and stack self-contained section components. Sections don't receive props from pages — they manage their own data internally.

### Data Layer
**Marketing pages**: All content hardcoded in:
- `src/lib/constants.ts` — Services, industries, metrics, booking info
- `src/lib/data/` — Project case study data
- `src/lib/types/` — TypeScript interfaces (Project, TechStack, Screenshot)
- `src/lib/hooks/` — Custom hooks (useBookingForm, useCarouselNavigation, useScrollAnimation, useSwipeGesture)

**Authenticated app** (`/app/*`): Supabase backend:
- `src/lib/supabase.ts` — Supabase client (anon key)
- `supabase/functions/server/` — Hono edge functions (Gemini AI integration)
- Tables: `wizard_sessions`, `business_profiles`, `diagnostics`, `recommendations`
- Auth: Supabase email/password + Google OAuth
- Dashboard: fetches live AI insights on mount → loading skeleton → merges with static data

- **Section pattern**: `max-w-[1200px] mx-auto px-6` for consistent content width
- **Navigation**: Pill-shaped container with sticky positioning
